# フロントエンド詳細設計（ログイン後アプリ）

LP は別設計とする。本ドキュメントは**ログイン後のウェブアプリ**の画面構成・UI・ナビゲーションを定義する。

---

## 技術方針

| 項目 | 採用技術・方針 |
|------|----------------|
| **3D** | **Three.js** で広場シーン・キャラクターを実装する |
| **アプリ形態** | **PWA（Progressive Web App）** とし、オフライン対応・ホーム画面への追加・プッシュ通知など、最先端の Web アプリ体験を提供する |

- 3D は React 等から **Three.js** を直接利用する形で実装（必要に応じて React Three Fiber 等のラッパーも可）。
- PWA として **manifest.json**（アプリ名・アイコン・テーマ色）、**Service Worker**（キャッシュ・オフライン）、必要に応じてインストール促進 UI を用意する。

---

## 1. 全体の流れ

- ログイン成功 → **ホーム画面**に遷移
- ホーム画面がアプリの起点。ここから検索・広場・商品一覧・アカウント設定へ遷移する

---

## 2. レイアウト共通要素

### 2.1 ヘッダー

- 画面上部に固定（またはスクロールで追従）
- **右上**に**ハンバーガーメニュー**（アイコン＋クリックで開閉）
- ヘッダー内の**右下**側に、広場モードへの**切り替えボタン**を配置（ハンバーガーとは別。通常ビュー ⇔ 広場ビューのトグル）

※「ヘッダーの右下」＝ヘッダー右側エリアの下、またはハンバーガーの近くに切り替えボタンがあるイメージ。実装時は「検索画面と広場を切り替えるボタン」として右上エリアにまとめてもよい。

### 2.2 ハンバーガーメニュー（開いたときの項目）

| 項目 | 説明 | 遷移先 |
|------|------|--------|
| ホーム | ログイン後のトップ | ホーム画面 |
| 広場 | 3D 広場ビュー | 広場画面（検索窓＋3D シーン） |
| 商品一覧 | 商品を一覧で見る | 商品一覧画面 |
| アカウント設定 | プロファイル・設定 | アカウント設定画面 |

- メニューはドロワーまたはドロップダウンで表示
- 選択した項目に応じて画面を切り替える（SPA 想定）

---

## 3. 画面別仕様

### 3.1 ホーム画面

- **役割**: ログイン後の最初の画面。アプリの入口
- **構成**:
  - ウェルカムメッセージやショートカット（検索へ・商品一覧へ・広場へ）などを配置
  - 必要に応じておすすめ商品や直近の提案へのリンクを表示
- **ルート例**: `/app` または `/app/home`

---

### 3.2 検索画面（メインの利用画面）

- **役割**: 検索窓で必要なものを検索し、その下に商品を一覧表示する
- **構成**:
  - **上部**: **検索窓**（テキスト入力）。ここでキーワードや欲しいものを入力
  - **下部**: **コンテナ**のように商品が並ぶ
    - カード形式で商品をグリッド表示（画像・名前・価格・割引・賞味期限など）
    - 検索結果または「人気・おすすめ」などで一覧を表示
- **ヘッダー**: 右上にハンバーガーメニュー、その近く（ヘッダー右下側）に**「広場」への切り替えボタン**
- **ルート例**: `/app/search` または `/app` を検索メインにする

---

### 3.3 広場画面（3D ビュー）

- **役割**: 3D アニメーションで「簡易的な人間がいる広場」を表現し、同じ検索窓で検索すると店舗情報がポップアップで出てくる体験を提供する
- **構成**:
  - **上部**: **検索窓**（検索画面と同じ。同じ入力欄で「必要なもの」を検索）
  - **メインエリア**: **3D シーン**
    - 簡易的な**人間キャラクター**がいる**広場**のような空間
    - 3D アニメーションで動きや雰囲気を表現（ライブラリは別途選定。例: Three.js / React Three Fiber 等）
  - **検索実行時**: 検索窓で検索すると**ポップアップ**（モーダルまたはオーバーレイ）が表示される
    - ポップアップ内容: **各店舗の情報**（店舗名・場所・取り扱い商品の概要など）
    - 複数店舗がある場合は一覧で出し、選択すると詳細や商品へ誘導できるようにする
- **切り替え**: ヘッダー右下のボタンで「検索画面（通常）」⇔「広場画面」を切り替え
- **ルート例**: `/app/plaza`

---

### 3.4 商品一覧画面

- **役割**: 検索に依存せず、商品を一覧で見る
- **構成**: 商品カードのグリッド。在庫・割引・賞味期限などを表示。architecture の「商品選択画面」に準拠
- **ルート例**: `/app/products`

---

### 3.5 アカウント設定画面

- **役割**: プロファイル（表示名・アレルギー情報など）の編集、ログアウト
- **構成**: フォームとボタン。GET /api/users/me で取得した情報を表示・更新
- **ルート例**: `/app/account` または `/app/settings`

---

## 4. ナビゲーション・切り替えの整理

| 操作 | 挙動 |
|------|------|
| ハンバーガー → ホーム | ホーム画面を表示 |
| ハンバーガー → 広場 | 広場画面を表示（検索窓＋3D 広場） |
| ハンバーガー → 商品一覧 | 商品一覧画面を表示 |
| ハンバーガー → アカウント設定 | アカウント設定画面を表示 |
| ヘッダー右下の切り替えボタン | 現在が「検索画面」なら「広場」へ、「広場」なら「検索画面」へトグル |

---

## 5. ルーティング案

```text
/app                 # ホーム（ログイン後トップ）
/app/search          # 検索画面（検索窓＋商品コンテナ）
/app/plaza           # 広場画面（検索窓＋3D 広場＋検索で店舗ポップアップ）
/app/products        # 商品一覧
/app/account         # アカウント設定
```

- `/app` をホームにし、`/app/search` を検索メインにしてもよい。または `/app` を検索画面にしてホームを `/app/home` にしてもよい。

## 7. 技術メモ（3D・広場）

- **3D 広場**: **Three.js** でシーンを組み、簡易的な人間（GLB/GLTF または簡易ジオメトリ）と広場のオブジェクトを配置する。React から利用する場合は `react-three-fiber` 等のラッパーを使用してもよい。
- **検索と店舗ポップアップ**: 検索窓の入力で API（商品・店舗検索）を呼び出し、返ってきた店舗情報をポップアップで表示する。
- **パフォーマンス**: モバイル・PWA でも快適に動くよう、3D は軽量なアセット・描画設定（解像度・シャドウ等）にする。

---

## 8. PWA 要件

- **manifest.json**: アプリ名、ショートカット名、アイコン（複数解像度）、`display: standalone`（または `minimal-ui`）、テーマ色・背景色を定義する。
- **Service Worker**: 静的アセットや API レスポンスのキャッシュ方針を決め、オフライン時も基本画面が表示できるようにする（任意: オフライン用フォールバックページ）。
- **HTTPS**: PWA は HTTPS 必須。Vercel デプロイで対応。
- **インストール**: 「ホーム画面に追加」や「アプリとしてインストール」を促す UI を必要に応じて実装する。
