# モバイルファーストのレイアウト設計

## 考え方

**モバイルファースト** = まず小さい画面（スマホ）用にレイアウト・余白・フォントサイズを決め、そのあと `sm:` / `md:` / `lg:` でタブレット・PC用に「足していく」設計です。

- **デスクトップファースト**: 大きな画面用に作ってから `max-width` で狭くする → モバイルで詰まりやすい
- **モバイルファースト**: 狭い画面で読みやすくしてから、余裕がある幅で並びや余白を変える → どの幅でも崩れにくい

## 手順（3ステップ）

### 1. モバイルをデフォルトにする

- 余白: `p-4`, `gap-4`, `py-6` など、小さい画面で十分な値から始める
- 文字: `text-base`（16px）以上にして、タップしやすい高さ（最低 44px）を意識する
- レイアウト: **1カラム**が基本。`flex flex-col` や `block` で縦に並べる

### 2. 幅に応じて「上書き」する（Tailwind）

Tailwind は **デフォルト = モバイル**、プレフィックスで **幅がそれ以上のとき** の指定になります。

| プレフィックス | 幅の目安 |
|----------------|----------|
| （なし）       | 0px 以上 = 全幅（モバイル含む） |
| `sm:`          | 640px 以上 |
| `md:`          | 768px 以上 |
| `lg:`          | 1024px 以上 |
| `xl:`          | 1280px 以上 |

例:

- モバイル: 常に縦並び → `flex flex-col gap-4`
- タブレット以上: 横並びに → `md:flex-row md:flex-wrap`
- モバイル: 余白 1rem → `p-4`
- PC: 余白を広く → `md:p-8` または `lg:px-12`

### 3. コンテンツの優先順位

- モバイルでは **1画面に載る量を少なめ** に（ヒーロー・CTA・重要メッセージを上に）
- 長いリストは **スクロール** で見せるか、タブ・アコーディオンで隠す
- ナビは **ハンバーガーメニュー** などで折りたたみ、PC では `md:flex` で横並びにする

## LP での具体例

- **ヒーロー吹き出し**: モバイルでは `flex-col` で上から並べ、`md:flex-row` で左右に分ける
- **メリット / 使い方**: モバイルは `grid-cols-1` または `grid-cols-2`、`sm:grid-cols-3` で段々増やす
- **最大幅**: モバイルは `w-full`、テキストブロックは `max-w-[560px] mx-auto` で読みやすさを確保
- **フォント**: `text-[clamp(1rem,2.5vw,1.25rem)]` で画面幅に応じてスケール
- **タップ領域**: ボタンは `min-h-[44px]` や `py-3` で指で押しやすくする

- [ ] 横並びは `md:` 以降でだけ使っているか

- [ ] ボタン・リンクが 44px 以上の高さか
